Agentic Reproducibility Engine1
Problem

Reproducibility evidence is scattered.

Important details live across PDFs, arXiv, DOI metadata, GitHub repositories, datasets, and informal notes.

Manual audits are slow, inconsistent, and often hide uncertainty.

Problem2
Solution

A visible multi-agent audit workflow.

  • Submit a paper or paper reference.
  • Choose autonomy level.
  • Watch trace, tools, evidence, objections, and report provenance.
Solution3
Agentic Workflow

Separate agents, separate artifacts.

Plan audit scope
Read paper claims
Retrieve evidence
Score reproducibility
Plan replication
Critique and report
Workflow4
AMD Architecture

Static frontend, AMD-hosted backend.

  • Hugging Face Static Space hosts the UI.
  • FastAPI agent backend runs on AMD Developer Cloud.
  • vLLM serves Qwen/Qwen3.5-27B on ROCm.
Architecture5
Evidence Tools

Live resolvers, fail-closed decisions.

  • arXiv, DOI, GitHub, and dataset resolver tools.
  • Missing evidence becomes an explicit gap.
  • The verifier can degrade the decision.
Evidence6
Demo Flow

Judges inspect the work, not just the answer.

Submit paper -> autonomy slider -> live trace -> evidence panel -> verifier objections -> final report.

Demo7
Evaluation

Agent behavior is tested.

  • Prompts, contexts, model artifacts, and tool calls are graded.
  • Resolver coverage and verifier behavior are checked.
  • Placeholder evidence is explicitly banned.
Evals8
Business Value

Faster first-pass reproducibility audits.

For research labs, ML teams, funders, reviewers, benchmark maintainers, and enterprise AI teams evaluating research-derived systems.
